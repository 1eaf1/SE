### 2.2 Настройка
```
git config --global user.name "username"
git config --global user.email "email"
git --list
git config --global core.editor "code --wait"
```
### Результат
![Меню](https://github.com/1eaf1/SE/blob/Тема_1/pic/2.2%20Настройка.png)
## Выводы
В данном номере происходит настройка/авторизация в Git при помощи команд ```git config --global user.name "имя пользователя"``` и ```git config --global user.email "почта пользователя"```. Проверка установки/настроек осущетсвляется с помощью команды ```git --list```. Помимо авторизации в git можно настроить редактор для сообщений коммитов при помощи команды ```git config --global core.editor "code --wait"``` - в данном случае происходит смена редактора Vim на Visual Studio Code.

## Лабораторная работа №9
### 2.9 Удаление файлов, веток, локальных и удалённых репозиториев
Команда ```git rm имя_файла``` позволяет полностью удалить файл из репозитория и рабочего каталога.
Команда ```git rm --cached имя_файла``` позволяет удалить файл из репозитория, но оставляет его в рабочем каталоге.
![2.91](https://github.com/1eaf1/SE/blob/Тема_1/pic/2.9.1%20Удаление.png)
При помощи команд ```git branch -d имя_ветки``` и ```git branch -D имя_ветки``` происходит удаление локальных веток. В первом случае происходит "мягкое" удаление - проверка слияние ветки, во втором случае принудительное удаление ветки без проверки.
![2.92](https://github.com/1eaf1/SE/blob/Тема_1/pic/2.9.2%20Удаление.png)
```git push origin --delete имя_удалённой_ветки``` - позволяет удалить ветку в удалённом репозитории
![2.93](https://github.com/1eaf1/SE/blob/Тема_1/pic/2.9.3%20Удаление.png)
## Вывод
В ходе выполнения данного пункта при помощи команд Git получилось удалить файлы и ветки из локального репозитория, и удалить ветку в удалённом репозитории.

## Лабораторная работа №12
### 2.12 Возвращение к предыдущему коммиту
```
git reset --hard HEAD^
git reset --soft HEAD^
git reset --hard коммит_хэш
```
![2.121](https://github.com/1eaf1/SE/blob/Тема_1/pic/2.12.1%20Возвращение%20к%20предыдущему%20коммиту.png)
![2.122](https://github.com/1eaf1/SE/blob/Тема_1/pic/2.12.2%20Возвращение%20к%20предыдущему%20коммиту.png)

## Выводы
```git reset --hard HEAD^``` осуществляет полное удаление последнего коммита с потерей изменений. --soft же удаляет коммит, но сохраняет изменения.
```git reset --hard коммит_хэш``` осуществляет сброс до коммита, которому присущ данный хэш.
