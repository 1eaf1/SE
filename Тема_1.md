## Лабораторная работа №2
### 2.2 Настройка
```git config --global user.name "username"```
```git config --global user.email "email"```
```git --list```
```git config --global core.editor "code --wait"```
### Результат
![Меню](https://github.com/1eaf1/SE/blob/Тема_1/pic/2.2%20Настройка.png)
## Выводы
В данном номере происходит настройка/авторизация в Git при помощи команд ```git config --global user.name "имя пользователя"``` и ```git config --global user.email "почта пользователя"```. Проверка установки/настроек осущетсвляется с помощью команды ```git --list```. Помимо авторизации в git можно настроить редактор для сообщений коммитов при помощи команды ```git config --global core.editor "code --wait"``` - в данном случае происходит смена редактора Vim на Visual Studio Code.

## Лабораторная работа №6
### 2.6 Подключение к удалённому репозиторию
```git remote add origin "URL_удалённого_репозитория"
git push origin master
git pull origin master
git pull --rebase origin master
git stash
git stash "Описание изменений"
git stash apply
git stash pop
```

![2.61](https://github.com/1eaf1/SE/blob/Тема_1/pic/2.6.3%20Подключение%20к%20удалённому%20репозиторию.png)

![2.63](https://github.com/1eaf1/SE/blob/Тема_1/pic/2.6.3%20Подключение%20к%20удалённому%20репозиторию.png)

## Выводы
Удалось произвести подключение к удалённому репозиторию на GitHub, а также загрузить данные с локального репозитория на удалённый при помощи ```git push origin master``` и выгрузить данные с удалённого репозитория на локальный при помощи команды ```git pull origin master```. При помощи ```git stash``` и ```git stash apply``` получилось зафиксировать несохранённые изменения, а затем вернуть их обратно.
